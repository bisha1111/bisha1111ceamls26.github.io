what_i_learned: |
  Today, I continued working on the individual model-training task we started earlier
  this week. I was able to train a model using my cleaned dataset, but the results were
  not as strong as I expected. Instead of immediately retraining it, I spent time
  checking the pipeline to find out what might have affected the model's performance.

  I reviewed how the images were being loaded and processed, checked the preprocessing
  steps, and made sure the data was flowing through the model correctly. I also looked
  over the model architecture to confirm that it matched the setup we intended to use.
  This helped me understand that weak results can come from different parts of the
  pipeline, not only from the model itself.

  Later in the day, our team began working on the application side of the DermaBridge
  project. We reviewed the files, requirements, and dependencies needed to run the skin
  lesion diagnosis application that our mentor has been developing. We also started
  setting up the GitHub repository in VS Code so that we can run the application locally
  and begin making changes.

  We also took a short break from coding and played a group game of Imposter. It was a
  fun way to relax, recharge, and spend time with the other participants before
  returning to our work.

blockers: |
  My biggest challenge was setting up the GitHub repository in VS Code. Many of the
  instructions were written for macOS, but I am working on a Windows computer. Some of
  the commands, file paths, and setup steps did not work the same way, which caused
  several configuration problems.

  I also had difficulty understanding why my model did not perform as well as expected.
  There are several possible causes, so I had to check the data, preprocessing steps,
  and model settings carefully. I am still working through these issues and hope to
  resolve them so I can fully begin working on the application.

reflection: |
  Today reminded me that machine learning involves a lot of debugging and troubleshooting.
  Training a model is only one part of the process. When the results are disappointing,
  it is important to slow down, examine each step, and understand what may have gone
  wrong before making changes.

  Even though the model performance was not what I hoped for, I still learned a lot from
  reviewing the pipeline. It helped me better understand how the data, preprocessing,
  architecture, and training process all work together.

  I am also excited that we are beginning application development. After spending weeks
  working on research, datasets, and models, it feels rewarding to start working on the
  tool that will bring the different parts of DermaBridge together. I look forward to
  fixing the remaining setup issues and contributing more to the application.
