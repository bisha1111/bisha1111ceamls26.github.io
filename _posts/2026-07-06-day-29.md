what_i_learned: |
  Today, our team focused on completing and improving the methodology and exploratory data analysis documentation for our DermaBridge skin cancer detection project. In the morning, we worked in Overleaf to finish our assigned methodology subsections. Each person focused on making their section clearer, more detailed, and more connected to the overall purpose of the research. My responsibility was connected to the preprocessing pipeline, so I continued refining how we explained the steps used to prepare the image and metadata before training.

  I worked on describing how the images are converted to RGB format, resized to a consistent shape, normalized, and prepared for machine learning models. I also reviewed how missing or inconsistent metadata should be handled so that the dataset can be used more reliably. This helped me understand that methodology writing is not only about listing steps. It has to explain why each step matters and how it supports reproducibility. If another researcher reads our paper, they should be able to understand what we did and repeat the process.

  We also reviewed our exploratory data analysis findings from the datasets we studied last week. We documented important details such as dataset name, number of images, diagnosis categories, class distribution, skin-tone coverage, and whether the dataset could fit into our current pipeline. I learned that a dataset should not be judged only by its size. A large dataset can still be limited if it has poor labels, missing metadata, weak diversity, or categories that do not match the rest of the project.

  A major part of today was learning how the ITA Fitzpatrick method can support fairness analysis. We used the ITA formula to create a `fitz_scale` column from the cleaned balanced metadata file. This gave us a more structured way to estimate skin-tone categories instead of relying only on subjective labels like `skin_tone` or `complexion`. We also compared the new `fitz_scale` column with the existing skin-tone information to see where they agreed or differed.

  This was important because our project is focused on building a skin lesion screening system that is more explainable and fair across different skin tones. Many medical AI datasets underrepresent darker skin tones, which can affect model performance and trust. By using ITA-based Fitzpatrick analysis, we can better document skin-tone representation and identify possible fairness gaps before model evaluation.

blockers: |
  One challenge today was making the methodology section detailed enough without making it too long or repetitive. There are many preprocessing steps, but each one needs to be explained clearly and connected to the research goal. It was sometimes difficult to decide which details belonged in the main methodology section and which details should be summarized more briefly.

  Another blocker was comparing datasets that reported skin-tone information in different ways. Some datasets used direct skin-tone labels, some used complexion descriptions, and some did not provide enough information at all. This made it harder to create one consistent fairness analysis across all datasets.

  Working with the ITA formula also required careful attention because the result depends on image color information and the correct extraction of pixel values. If the calculation is not done carefully, the `fitz_scale` column may not accurately reflect the intended skin-tone category. We had to think about how technical choices in preprocessing can affect the fairness analysis later.

  Another challenge was connecting the EDA findings to the research paper in a meaningful way. It is not enough to say that one dataset has more images than another. We also need to explain how class imbalance, missing demographic information, and skin-tone coverage can affect model reliability, bias, and generalization.

reflection: |
  Today showed me that strong research depends on both technical work and clear documentation. Coding, cleaning data, and building models are important, but the project becomes more trustworthy when every step is explained in a way that another person can understand and verify. Methodology writing forces us to slow down and think carefully about the decisions we made.

  I also learned that exploratory data analysis is one of the most important parts of a machine learning project. EDA helps reveal what the dataset actually contains before we trust a model trained on it. It shows class imbalance, missing values, weak labels, and fairness concerns that may not be obvious at first. This is especially important in healthcare AI, where mistakes can affect real people.

  The ITA Fitzpatrick analysis helped me connect fairness to actual data instead of only discussing it as an idea. Since DermaBridge is meant to support more trustworthy skin lesion screening, we need to understand whether our datasets include enough representation across different skin tones. This matters deeply because people with darker skin tones have often been underrepresented in dermatology datasets, and that can lead to unequal model performance.

  By the end of the day, I felt that our project became more organized and research-ready. The methodology section was clearer, the EDA documentation was stronger, and the ITA Fitzpatrick work gave us a better foundation for discussing fairness. I also understood more clearly that building an AI system for skin cancer detection is not just about getting high accuracy. It is about making sure the data, methods, and explanations are reliable, fair, and useful for the communities the system is meant to serve.
